After learning fundamentals, this section will explore building a real-world application with the following features.

 - A front-end app with 
    - custom components
    - authentication
    - routes and access control
    - data fetching
    - analytics & performance optimization
 - An serverless backend with
    - Open API spec + Serverless functions
    - Static Site + Cloud hosting
    - NoSQL database + Serverless usage
 - Automated workflows for 
    - Build/deploy (GitHub Actions)
    - E2E Testing (Playwright)

## Development Plan

 1. Deconstruct existing example to understand best practices. Ex: [Tastejs/Movies](https://tastejs.com/movies/) - built with [SvelteKit](https://github.com/tastejs/sveltekit-movie-app). 
 2. Identify a comparable application (similar flows, API, routes) - write end-to-end test specification.
 3. Implement it using test-driven development - with real-world examples providing reference code.
